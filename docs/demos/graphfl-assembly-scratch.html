Graph-FL Design Lab — 그래프를 만들고, 비교하고, 설명하는 실험 프레임워크

Graph-FL 방법은 client representation, relation/topology, edge weight, aggregation target 중 일부를 바꾸며 발전해왔습니다. 이 데모는 그 축을 repo의 실제 이름(graph_source, graph_mode, aggregation_target, …)으로 조립하고, 같은 control과 evidence bundle로 비교 가능한 CLI + JSON을 만듭니다. 새 그래프도 레지스트리에 이름을 등록하면 같은 실험 흐름에 얹을 수 있습니다.

Graph-FL gain을 accuracy 하나로 끝내지 않고, relation/topology, smoothing, dominance, optimizer 효과를 같은 실험에서 분해한다.
assemble graph → matched controls → evidence bundle → claim boundary
조립 representation·relation·topology·aggregation을 블록처럼 고른다.
대조 random·shuffled·uniform·graph-free로 비교 조건을 맞춘다.
근거 accuracy/loss와 alignment, DI/N_eff, LOO, gap을 한 결과에 묶는다.
해석 성능 상승이 relation, topology, smoothing 중 어디에서 오는지 좁힌다.
기본 경로: client update → similarity/RBF → kNN graph → graph_filtered_update 판단 지표: accuracy/loss · real-control gap · graph-free gap 기전 지표: alignment · DI/N_eff · LOO · smoothness/spectral diagnostics

한 열은 하나의 비교 가능한 개입입니다. 주황은 트랙·데이터·규모 같은 공통 실행환경이고, 각 열은 같은 조건에서 하나의 graph intervention을 바꾼 비교 단위입니다. baseline은 1블록으로 끝나고, Graph-FL은 graph_source → relation cue → graph_mode → aggregation → optional control 순서로 조립됩니다. relation cue는 발표용 해석 축이고, 실제 config에는 repo가 받는 graph_mode만 기록됩니다. 가능한 한 같은 pipeline을 공유하고, relation 의미·identity 대응·edge 구조·graph-free correction처럼 검증하려는 요인만 바꿉니다. Preflight/suite_rows/LOO/Evidence/counterfactual은 조립 블록이 아니라 실행 관제 옵션과 결과 아티팩트로 기록됩니다.
